
# PERMANENT PROJECT RULE — ENTERPRISE ARCHITECTURE & MODULARIZATION STANDARD

Everything must be organized by feature and responsibility. Deep tree structure. Every major feature, page, dashboard, reusable component, layout, service, API module, hook, utility, animation, context, constant, validation, mock data file, asset group must have its own folder.

Apply Feature-First Architecture, Component Architecture, UI Element Isolation, Backend Architecture (Apps, Views, Models, Serializers, etc), Database Architecture.

Clean Code Standards: SOLID Principles, DRY, KISS, Separation of Concerns.

File Size Standards: Components below ~150 lines, Functions below ~50 lines, Hooks below ~100 lines.

No empty folders/files. Descriptive naming.
